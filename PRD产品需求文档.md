# Mina AI舆情巡航系统 — PRD 产品需求文档

> 版本：v1.0  
> 更新时间：2026-04-16  
> 产品定位：AI 驱动的多源舆情分析与报告生成系统

---

## 一、产品概述

### 1.1 产品愿景
构建一个基于多 AI Agent 协同的舆情分析系统，能够自动从本地数据库、网页媒体、新闻源三路并发采集数据，通过 LLM 深度分析后生成专业舆情报告，帮助用户快速洞察舆论动态。

### 1.2 核心价值
- **多源数据融合**：本地数据库 + 网页媒体 + 新闻源三路并发采集
- **AI 深度分析**：5 个专业 AI Agent 协同工作，各司其职
- **秒级缓存响应**：相同查询 6 小时内秒出，模糊匹配 90 天内智能复用
- **专业报告输出**：6 种预设模板，自动选择最佳报告结构
- **实时进度可视化**：打字机终端 + 节点状态机，全程透明

### 1.3 目标用户
- 企业品牌公关团队
- 市场研究分析师
- 政府舆情监测部门
- 媒体内容运营团队

## 二、功能需求

### 2.1 核心功能模块

#### 模块 1：搜索与数据采集
**功能描述**：用户输入关键词后，系统自动启动三路并发采集引擎。

**子功能**：
1. **InsightEngine（本地数据库引擎）**
   - 从 MySQL/PostgreSQL 数据库查询历史新闻、话题、关联关系
   - 使用 Kimi-K2 进行深度分析与反思迭代（最多 3 轮）
   - 支持关键词智能优化（Qwen-Plus）
   - 支持 22 种语言情感分析
   - 使用 KMeans 聚类对搜索结果去重采样

2. **MediaEngine（网页媒体引擎）**
   - 通过 Bocha AI / Anspire AI 搜索网页内容
   - 支持多模态搜索（文本/图片/视频）
   - 使用 Gemini-2.5-Pro 进行媒体内容分析

3. **QueryEngine（新闻引擎）**
   - 通过 Tavily 搜索实时新闻（6 种搜索工具）
   - 使用 DeepSeek-Chat 进行新闻查询与时事研究

**交互流程**：
```
用户输入关键词 → 点击搜索
    ↓
前端切换到 Forum State（运算态）
    ↓
三个 Agent 节点开始呼吸动画（idle → active）
    ↓
打字机终端实时输出各引擎日志
    ↓
各引擎完成后节点标记为 ✅ 完成
```

**验收标准**：
- 三路引擎并发启动，总耗时不超过单引擎串行的 40%
- 每个引擎失败不影响其他引擎，至少有一个成功即可继续
- 日志输出延迟 < 500ms
- 支持 Token 预算熔断，超预算时带半成品数据继续

---

#### 模块 2：报告生成与编排
**功能描述**：ReportEngine 接收三路引擎的原始报告，自动选择模板、设计文档结构、分配字数预算、逐章生成最终报告。

**子功能**：
1. **模板自动选择**
   - 根据查询意图匹配 6 种预设模板：
     - 企业品牌声誉分析报告
     - 市场竞争格局舆情分析
     - 日常/定期舆情监测
     - 特定政策或行业动态
     - 社会公共热点事件
     - 突发事件与危机公关

2. **文档布局设计**
   - 自动生成章节结构（标题、层级、顺序）
   - 支持自定义模板（用户可上传）

3. **字数预算分配**
   - 按章节重要性分配字数预算
   - 确保报告篇幅合理（默认 5000-8000 字）

4. **章节内容生成**
   - 逐章节调用 Gemini-2.5-Pro 生成正文
   - 支持引用三路引擎的原始数据
   - 自动插入图表（词云、情感分布、时间线）

5. **多格式输出**
   - HTML（默认，带交互式 ToC）
   - PDF（服务端 WeasyPrint / 客户端 html2canvas + jsPDF）
   - Markdown

**交互流程**：
```
三路引擎完成 → ReportEngine 启动
    ↓
终端输出：✍️ 三路数据汇总完毕，ReportEngine 正在极速撰写最终报告...
    ↓
逐章生成进度推送（SSE 流）
    ↓
报告生成完毕 → 自动弹出预览（iframe）
    ↓
用户可导出 PDF / Markdown / 下载 HTML
```

**验收标准**：
- 报告生成耗时 < 60 秒（5000 字报告）
- HTML 报告支持目录跳转、章节折叠
- PDF 导出布局无错位、中文字体正常显示
- 支持 Token 预算熔断，超预算时生成截断报告

---

#### 模块 3：报告缓存与历史管理
**功能描述**：相同查询在 TTL 内直接返回缓存，避免重复计算；支持模糊匹配，相似查询智能复用。

**子功能**：
1. **精确缓存匹配**
   - 查询词归一化（去空格、转小写）
   - TTL 默认 6 小时（可配置）
   - 命中后 0 秒返回，0 Token 消耗

2. **模糊缓存匹配**
   - 90 天内历史查询相似度 ≥ 0.92 视为重复
   - 使用 difflib.SequenceMatcher 计算相似度
   - 最多扫描 200 条候选记录

3. **历史报告列表**
   - 按时间倒序展示最近 50 条报告
   - 显示查询词、生成时间、报告路径
   - 支持点击直接预览

4. **缓存失效处理**
   - 报告文件被删除时自动清理缓存记录
   - 支持手动清理超过 30 天的旧缓存

**交互流程**：
```
用户提交搜索 → 后端检查缓存
    ↓
命中缓存？
    ├── 是 → 终端输出：命中缓存，直接返回历史报告
    │         ↓
    │      0.3 秒内弹出报告
    │
    └── 否 → 启动完整流水线
              ↓
           生成完毕后写入缓存
```

**验收标准**：
- 缓存命中率 > 30%（生产环境）
- 缓存查询耗时 < 100ms
- 模糊匹配准确率 > 95%（人工抽检）
- 缓存数据库大小 < 500MB（1000 条报告）

---

#### 模块 4：实时进度可视化
**功能描述**：通过 Forum State 运算态界面，实时展示三个 Agent 节点的工作状态和日志输出。

**子功能**：
1. **节点状态机**
   - 三个节点（Query / Media / Insight）
   - 状态：idle（呼吸动画）→ active（激活动画）→ done（完成标记）
   - SVG 连接线 + 脉冲动画

2. **打字机终端**
   - 实时输出各引擎日志（带时间戳、Agent 标签、颜色区分）
   - 打字机速度：18ms/字符（长文本自动加速）
   - 最多保留 60 行日志（超出后移除最旧的）

3. **阶段消息映射**
   - 后端 stage/status 自动映射为中文消息
   - 例如：`query_running` → "🔍 [全网实时资讯] 正在提取核心观点，请稍候..."

**交互流程**：
```
搜索提交 → Spotlight 收缩
    ↓
Forum State 展开（350ms 延迟）
    ↓
三个节点进入 idle 呼吸态
    ↓
后端推送日志 → 打字机输出
    ↓
各引擎完成 → 节点标记 ✅
    ↓
报告生成完毕 → 自动弹出预览
```

**验收标准**：
- 节点动画流畅（60fps）
- 日志输出无卡顿、无丢失
- 打字机效果自然（不过快/过慢）
- 支持窗口 resize 时 SVG 连接线自动重绘

---

#### 模块 5：配置管理
**功能描述**：支持运行时查看和修改系统配置，无需重启服务。

**子功能**：
1. **配置查看**
   - 展示当前 LLM API Keys、搜索工具配置、数据库连接等
   - 敏感字段（密码、API Key）部分脱敏显示

2. **配置修改**
   - 支持在线修改配置项
   - 自动写入 `.env` 文件
   - 重载配置（无需重启 Flask）

3. **配置验证**
   - 修改前校验格式（URL、端口、布尔值等）
   - 修改后测试连接（数据库、LLM API）

**验收标准**：
- 配置修改后 < 3 秒生效
- 修改失败时回滚到旧配置
- 敏感字段脱敏显示（如 `sk-***abc`）

---

#### 模块 6：论坛讨论模拟（ForumEngine）
**功能描述**：模拟多个 AI Agent 进行论坛讨论，碰撞观点，生成更全面的分析视角。

**子功能**：
1. **论坛主持人**
   - 使用 Qwen-Plus 作为主持人
   - 引导讨论方向、总结观点

2. **多 Agent 讨论**
   - 三个引擎的结果作为不同"发言人"
   - 模拟辩论、补充、质疑

3. **讨论日志**
   - 实时输出讨论内容
   - 支持分页查看历史讨论

**验收标准**：
- 讨论轮次 ≥ 3 轮
- 讨论内容有逻辑连贯性
- 日志输出格式统一

### 2.2 非功能需求

#### 性能要求
| 指标 | 目标值 |
|------|--------|
| 搜索流水线总耗时 | < 120 秒（三路并发） |
| 报告生成耗时 | < 60 秒（5000 字） |
| 缓存命中响应时间 | < 0.5 秒 |
| 并发用户数 | ≥ 10（单机） |
| 数据库查询响应 | < 2 秒（复杂 JOIN） |

#### 可靠性要求
- 单引擎失败不影响整体流程（至少 1/3 成功即可）
- Token 预算熔断后带半成品数据继续
- 缓存失败不影响正常流水线
- 报告文件丢失时自动清理缓存记录

#### 可用性要求
- 系统 7×24 小时运行
- 支持 Docker 一键部署
- 支持数据库迁移（MySQL ↔ PostgreSQL）

#### 安全性要求
- API Key 加密存储（`.env` 文件权限 600）
- 敏感配置脱敏显示
- SQL 注入防护（使用参数化查询）
- XSS 防护（HTML 输出转义）

### 4.2 数据流
```
用户输入关键词
    ↓
POST /api/search
    ↓
runner.run_pipeline()
    ├── 检查缓存（精确 + 模糊匹配）
    │   └── 命中 → 直接返回（0 秒）
    │
    └── 未命中 → 三路并发采集
        ├── InsightEngine → MindSpider DB → Kimi-K2
        ├── MediaEngine → Bocha/Anspire → Gemini-2.5-Pro
        └── QueryEngine → Tavily → DeepSeek-Chat
            ↓
        三路结果汇总
            ↓
        ReportEngine.generate_report()
            ├── 模板选择
            ├── 文档布局设计
            ├── 字数预算分配
            └── 逐章生成（Gemini-2.5-Pro）
                ↓
            HTML 报告 + IR 中间表示
                ↓
            写入 SQLite 缓存
                ↓
            返回前端（Socket.IO 推送）
                ↓
            自动弹出报告预览
```

## 六、风险与应对

### 风险 1：LLM API 限流
**影响**：报告生成失败或耗时过长  
**应对**：
- 实现 Token 预算熔断，超预算时生成截断报告
- 支持多 LLM 提供商切换（Kimi / Gemini / DeepSeek）
- 缓存机制减少重复调用

### 风险 2：WeasyPrint PDF 导出卡死
**影响**：PDF 导出功能不可用  
**应对**：
- 已知问题，已在 CLAUDE.md 中记录
- 提供客户端 PDF 导出备选方案（html2canvas + jsPDF）
- 考虑替换为 Playwright PDF 导出

### 风险 3：数据库查询性能瓶颈
**影响**：InsightEngine 采集耗时过长  
**应对**：
- 优化 SQL 查询（添加索引、减少 JOIN）
- 使用异步数据库连接池（asyncpg / aiomysql）
- 限制查询结果数量（默认 TOP 1000）

### 风险 4：缓存数据库膨胀
**影响**：磁盘空间不足  
**应对**：
- 自动清理 30 天以上旧缓存
- 限制缓存条目数量（最多 1000 条）
- 定期压缩 SQLite 数据库（VACUUM）
